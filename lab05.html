AI-Assisted Development: Portfolio Enhancement & Python Generated Code

Experiment #1: Portfolio Enhancement

Experiment Overview

For this experiment, I used AI to enhance my portfolio and user experience by generating new code to add a dark model toggle and improve the overall visual design of my portfolio. The feature allows users to customize their experience by switching between light and dark modes, improving accessibility and user comfort. The AI-generated code was integrated into my existing HTML and CSS files to create a seamless dark mode toggle that remembers user preferences. I understood what the AI generated code did, but I had to do some research to fully understand how it functioned. I'm still gaining confidence with HTML web design, so this experiment, and the research that accompanied it, was extremely helpful in deepening my understanding of HTML and how it can be used to customize user interfaces.

Challenges and Solutions

The first output AI generated set the toggle and menu as two separate elements. This didn't present any issues when viewing the webpage in desktop view, but when the screen was resized to mobile view, the toggle and collapsed menu overlapped. I asked AI to adjust the code to include the toggle within the menu itself, which resolved the issue and created a neater interface for mobile users.

Experiment #2: Python Code Generation

Experiment Overview

For this experiment, I took a project I did in a previous class where I compared opioid dispensing rates and the Social Vulnerability Index (SVI) for different counties in DFW and used AI to complete the same tasks I completed in that project. The goal was to see how AI would compare to my human work in the original project, would it be more complicated? Would it make the process faster or more efficient?

Challenges and Solutions

Compared to my original project, the AI-generated code was not as efficient. Simple processes such as cleaning and merging the data took more steps than in the human generated code, and it required several prompts to achieve the desired outcome. Luckily, I had my original project to compare outputs, which helped me identify and correct issues within the AI-generated code when it came to cleaning.

Cleaning Data

My original code was more efficient and concise than the AI-generated code. I chose to isolate the DFW counties by FIPS codes in the opioid dispensing dataset and then I extracted FIPS codes, poverty data (EP_POV150), and unemployment data (EP_UNEMP) from the Social Vulnerability Index (SVI) dataset. My code did not require me to make a new DataFrame to isolate the DFW data, which made it very easy to keep track of my processes without juggling multiple DataFrames.

On the other hand, the AI-generated code was more complicated, required more steps to reach the same end-result, and was less readable. The AI-generated code created multiple DataFrames to isolate the data, which made it hard to follow when reviewing the generated code. As seen through the compared code, there was a much simpler approach that I had taken in my original code.

Merging Data

After AI had merged the datasets, it did not effectively clean the data. The AI-generated code did not properly handle duplicate county names or FIPS codes, which led to inconsistencies in the merged dataset. This was fixable, but required additional prompting of the AI.

Further Cleaning of AI-Merged Data

After continuously preparing the data, the AI-generated code was able to run an OLS regression model and provided the same results as my original project.

Critical Reflection

I believe vibe coding can help programmer development under the condition that the code is closely examined after being generated, going line by line, evaluating understanding, and being able to explain what exactly the generated code does. If vibe coding is used, but not closely evaluated once generated, there is no room for development as a programmer. Proper citation for AI-generated code within the code includes comments that explicit state which parts of the code are AI-generated. Outside of comments within code, proper citation does look like explicitly stating which part of the code is AI-generated, but also explaining how the generated code works, how much was initially understood, how understanding was developed, and how the code will assist in future programming. In my personal use, if I am struggling to figure out how to approach a certain task, I will use AI coding, but I will go line by line to understand how the code was developed, what it means, and learn from it. For coding tasks that have already been outlined in a lecture or have been done in previous projects, I will not use AI to assist coding because using AI would hinder my ability to understand through practice.
